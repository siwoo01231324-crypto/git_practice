# GIT Commit Message Rule

```
# 본문과 푸터는 선택 사항 입니다.

#######제목#######
# feat: 새로운 기능 추가, 기존의 기능을 요구 사항에 맞추어 수정
# fix: 기능에 대한 버그 수정
# style: 코드 스타일, 포맷팅에 대한 수정
# docs: 문서(주석) 수정
# build: 빌드 관련 수정
# chore: 패키지 매니저 수정, 그 외 기타 수정 ex) .gitignore
# ci: CI 관련 설정 수정
# refactor: 기능의 변화가 아닌 코드 리팩터링 ex) 변수 이름 변경
# test: 테스트 코드 추가/수정
# release: 버전 릴리즈

#######본문#######

#######푸터#######
# Resolves: feature-
# See also: hotfix-
# Related to: feature-

```

# 쉬어가는 이야기

## 리누스 토발즈가 말하는 Git 개발 비하인드

- Git은 리눅스 커널 개발에 사용되던 버전 관리 도구 BitKeeper의 사용이 중단되면서, 이를 대체하기 위해 개발됨
- 첫 번째 버전을 만드는 데 걸린 시간은 약 2주
- 사용자 단(side) 개발이라 커널보다 훨씬 쉽게 느껴졌다고 함
- 딸이 대학에 간 뒤, “아빠는 리눅스보다 Git으로 더 유명하대요”라는 말을 듣고 웃었다고 함
- Git에서 SHA-1을 사용한 이유는 보안을 위한 것이 아니라, 파일 손상 여부를 안정적으로 검증하기 위해서였다고 함
